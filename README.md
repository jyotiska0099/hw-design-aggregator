# hw-design-aggregator
A command-line tool that parses ARM CMSIS-SVD register-map files for MCUs (STM32, nRF52, etc.), models the hardware data using Pydantic, and generates C headers and Markdown reports via Jinja2 templates.

---

## Features

- Parse real-world `.svd` XML files using `lxml`
- Model device hierarchy: Device → Peripheral → Register → Field
- Auto-fetch SVD files by chip name from the cmsis-svd-stm32 repository
- Generate production-style C headers with `#define` macros for base addresses, register offsets, field positions, and bitmasks
- Generate structured Markdown reports with peripheral and register tables
- Local SVD caching to avoid redundant downloads
- Handles SVD quirks: `derivedFrom` inheritance, `bitRange` vs `bitOffset/bitWidth`, hex and decimal addresses

## Installation

```bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/hw-design-aggregator.git
cd hw-design-aggregator

# Create and activate virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

## Usage

### Parse a local SVD file

```bash
python3 main.py --svd svd_files/STM32F0x0.svd --output both
```

### Fetch a chip by name and generate outputs

```bash
python3 main.py --fetch STM32F103 --output both
```

### Generate only a C header

```bash
python3 main.py --fetch STM32F4x5 --output header
```

### Generate only a Markdown report

```bash
python3 main.py --svd svd_files/STM32F0x0.svd --output report
```

### CLI options

| Flag | Description | Default |
|---|---|---|
| `--svd FILE` | Path to a local .svd file | — |
| `--fetch CHIP` | Chip name to fetch (e.g. STM32F103) | — |
| `--output` | `header`, `report`, or `both` | `both` |
| `--out-dir DIR` | Root output directory | `output/` |
| `--templates DIR` | Jinja2 templates directory | `templates/` |
| `--svd-dir DIR` | Directory to cache SVD files | `svd_files/` |

## Output Examples

### C Header (`output/headers/STM32F0x0.h`)

```c
/* Auto-generated by Hardware Design Aggregator */
/* Device: STM32F0x0 */

#ifndef STM32F0X0_H
#define STM32F0X0_H

#include <stdint.h>

/* USART1 — Base Address: 0x40013800 */
#define USART1_BASE  (0x40013800UL)

/* CR1 — Control register 1 */
#define USART1_CR1_OFFSET  (0x000UL)
#define USART1_CR1  (*(volatile uint32_t *)(USART1_BASE + USART1_CR1_OFFSET))

#define USART1_CR1_UE_POS  (0UL)
#define USART1_CR1_UE_MSK  (0x1UL)
```

### Markdown Report (`output/reports/STM32F0x0.md`)

Includes a summary table of device properties, per-peripheral register tables, and per-register field breakdowns with bit positions, access types, and descriptions.

## SVD Format

CMSIS-SVD (System View Description) is an ARM-standardized XML format describing a microcontroller's memory-mapped peripheral registers. It is widely used in embedded toolchains (CMSIS, OpenOCD, pyOCD, SVD-Viewer in VS Code).

SVD files are open-source and available at:
- https://github.com/modm-io/cmsis-svd-stm32
- https://github.com/posborne/cmsis-svd
